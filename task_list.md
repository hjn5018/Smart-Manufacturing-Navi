# 작업 목록 (Task List)

## Phase 1: 프로젝트 초기화 및 가상 공정 환경(Mock) 구축
- [x] 프로젝트 리포지토리 생성 및 기본 폴더 구조(`frontend`, `backend`) 세팅.
- [x] 가상 데이터 모델링: 생산 라인, 설비, 센서, 생산 실적, 에러 로그 스키마 정의.
- [x] Mock 서버 API 개발: 가상 PLC, SCADA(실시간 제어/조회), MES(생산/불량 데이터), ERP 조회 엔드포인트 구현.
- [x] 프론트엔드 대시보드 기초 구현: 공정 상태 시각화 (차트, 게이지 등) 및 Mock 데이터 연동.

## Phase 2: AI 인터페이스 및 코어 로직 연동
- [ ] 프론트엔드 음성 인터페이스(Web Speech API 등 활용) 컴포넌트 개발: 마이크 입력, 음성 인식(STT).
- [ ] 백엔드 LLM 연동 모듈 개발: 음성 텍스트를 입력받아 의도(Intent) 추출 및 필요 액션 판단 로직 구현.
- [ ] 제어/조회 로직 맵핑: 분석된 의도에 따라 Mock 시스템(PLC, MES 등) API를 호출하고 결과를 가져오는 컨트롤러 작성.
- [ ] 응답 생성 및 TTS 적용: LLM을 통해 사용자 친화적인 응답 텍스트를 생성하고 프론트엔드에서 음성(TTS)으로 출력.

## Phase 3: RAG 및 온톨로지(지식 기반) 고도화
- [ ] 스마트 제조 도메인 온톨로지 설계 (설비-공정-오류 간의 관계 정의).
- [ ] 가상의 설비 매뉴얼, 유지보수 이력 문서 작성 및 Vector DB 임베딩 파이프라인 구축(RAG).
- [ ] 이상 탐지 및 해결 가이드 시나리오 연동: 에러 발생 시 RAG를 통해 조치 방법을 검색하여 답변하는 기능 추가.

## Phase 4: 시나리오 테스트 및 검증

- [ ] 시나리오 2 테스트: 생산 속도 조절, 설비 정지/재시작 제어 명령 및 상태 갱신 검증.
- [ ] 시나리오 3 테스트: 복합 에러 상황에 대한 로그 분석 및 RAG 기반 조치 방법 안내 검증.
